This tab demonstrates how The Ark's web services can be used to configure the pedigree module for a given study. A web service call is used to mimic the behaviour of The Ark's pedigree configuration window. The settings are:

  • Plot affected status? A yes/no toggle to switch on/off annotation of each subject's affected status.
  • Affected status field ? When "Plot affected status" is enabled, a subject custom field must be specified to provide the data for each subject's affected status. The custom field must be pre-defined in The Ark, and be of type CHARACTER with encoded values for "Yes" and "No", e.g. 1=Yes;0=No;
  • Plot date of birth? A yes/no toggle to switch on/off annotation of each subject's date of birth.
  • Plot age? A yes/no toggle to switch on/off annotation of each subject's current age.
  • Inbreeding allowed? A yes/no toggle to allow/disallow consanguineous relationships within a pedigree. This setting applies to both pedigree structures established using the pedigree module structures and visualisations. The default setting is "No". Once this setting has been enabled, it cannot later be disabled.

This example sets the configuration for the study Demo 2 within The Ark at http://demo.sphinx.org.au/ark
